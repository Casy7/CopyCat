# Технічні вимоги

### Функціональні вимоги (FR) 

Опис того, що система повинна робити для забезпечення заявленого бізнес-флоу — обміну текстом та невеликими файлами (до 20 МБ) між девайсами користувача.

| # | Вимога | Пріоритет |
| :--- | :--- | :--- |
| **FR-01** | Система дозволяє користувачу створити або приєднатися до кімнати (чату) шляхом введення 8-значного коду без необхідності реєстрації. | Must Have  |
| **FR-02** | Система дозволяє надсилати та відображати текстові повідомлення у вікні чату в режимі реального часу. | Must Have  |
| **FR-03** | Система дозволяє завантажувати файли та зображення розміром до 20 МБ і завантажувати їх на іншому девайсі. | Must Have  |
| **FR-04** | Система зберігає сесійні кукі на боці користувача для збереження доступу до чату навіть при перезавантаженні чи закритті сторінки. | Must Have  |
| **FR-05** | Система автоматично очищує дані (Garbage Collection), видаляючи кімнати та їхні файли, якщо в них не було активності більше ніж 24 години. | Should Have  |
| **FR-06** | Система генерує випадковий 8-значний код для створення нової кімнати за замовчуванням. | Nice to Have  |

### Нефункціональні вимоги (NFR) 

Обмеження системи, що визначають її архітектуру, продуктивність та безпеку. 

| NFR | Вимога | Обґрунтування |
| :--- | :--- | :--- |
| **Performance**  | Затримка доставки текстового повідомлення від одного клієнта до іншого не перевищує 500ms. | Необхідно для відчуття миттєвого «буфера обміну» в реальному часі. |
| **Security & Capacity**  | Жорстке обмеження розміру Multipart-запиту на бекенді (max-file-size: 20MB, max-request-size: 25MB). | Захист сервера від переповнення пам'яті (OOM) та DoS-атак через завантаження величезних файлів. |
| **Usability / Compatibility**  | 100% адаптивна верстка веб-інтерфейсу (Mobile First). | Користувачі гарантовано будуть відкривати застосунок зі смартфонів та ПК. |
| **Storage / Maintainability**  | Локальне сховище сервера під файли юзерів не перевищує ліміт у (наприклад) 5 GB. При досягненні ліміту старі файли видаляються примусово. | Уникнення падіння сервера через брак дискового простору, оскільки кімнати без авторизації можуть забивати пам'ять. |
